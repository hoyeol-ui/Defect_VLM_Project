1. Manufacturing scenario

Problem

데이터가 적은 제조 현장

  • 결함 이미지와 라벨이 부족함
  • annotation 비용이 큼
  • 어떤 이미지를 먼저 라벨링할지 결정해야 함
  • 설명 가능한 우선순위가 필요함

기존 제안

VLM consistency, DINO diversity, detector uncertainty를 조합해 GT-free active learning selector를 만든다.

2. What experiments taught us

Observed

Random은 약하지 않았다

  • balanced NEU에서는 Random도 class 6/6을 빠르게 커버
  • bbox count, multi-instance, entropy가 충분히 좋음
  • selector gain은 seed와 budget에 민감함
Stop

단조 proxy 가정 중단

  • box가 많으면 좋다 → 아님
  • box가 적으면 recall에 좋다 → 아님
  • uncertainty만 높으면 좋다 → 아님

3. Reframed hypothesis

New hypothesis

Selector가 아니라 audit-and-triage layer

GT-free signal은 모든 balanced dataset에서 Random을 이기는 마법의 selector가 아니라, Random이 약해지는 조건을 찾고 annotation queue를 보조하는 layer로 쓴다.

AL이 필요한 조건

  • rare defect / long-tail class
  • 중복 이미지가 많은 pool
  • domain shift
  • 극소 budget cold-start

4. New workflow decision

Gate first

Random baseline audit

먼저 Random curve와 데이터 감사를 수행한다. Random이 이미 강하면 active selector 대신 QA, augmentation, model scaling이 우선이다.

Use AL only if needed

조건부 GT-free AL

Random이 약한 조건에서만 VLM/DINO/detector signal을 labeling priority에 반영한다.
